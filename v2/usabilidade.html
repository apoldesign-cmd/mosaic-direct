01

Drawer lateral em vez de modal centralizado

Side Drawer · 560px · slide-in da direita
Problema
Modais centralizados bloqueiam o contexto da página e forçam o usuário a decorar o que estava vendo.
Ganho de UX
  • Contexto preservado — o operador continua vendo a tela de fundo enquanto interage com o drawer. Sem precisar decorar.
  • Ações sempre fixas no rodapé — botões primário/secundário ficam ancorados na barra inferior mesmo com scroll vertical interno. Ideal para drawers com conteúdo longo, onde o usuário não precisa rolar até o fim para confirmar.
  • Organização hierárquica do conteúdo — header (título + close), body (conteúdo rolável) e footer (ações) são camadas visualmente separadas, criando ritmo de leitura previsível.

02

Tabela zebrada (striped rows)

nth-child(even) { background: #f7f9f8 }
Problema
Em tabelas com muitas colunas, o olho perde a linha ao deslocar horizontalmente.
Ganho de UX
Faixas de cor ancoram o olhar e reduzem erros de leitura em tabelas densas. Diferença perceptual sem poluição visual.
WCAG 1.4.8 Material Design

03

Congelamento de coluna (sticky column)

position: sticky; left: 0
Problema
Com 24 colunas, o número do protocolo desaparecia no scroll. O usuário perdia a referência de qual linha estava lendo.
Ganho de UX
O identificador permanece visível em qualquer posição do scroll. Zero confusão de linha. Mesmo padrão de Google Sheets e Excel.

04

Agrupamento na linha (cell stacking)

.cell-stack { flex-direction: column }
Problema
Pares de campos relacionados ocupavam 2 colunas separadas, dobrando a largura da tabela.
Ganho de UX
~40% menos colunas visíveis. Informações relacionadas ficam juntas (proximidade). Leitura mais rápida em pares.
NN/g · Information density

Barra inferior fixa (sticky action bar)

.summary-footer { position: fixed; bottom: 0 }
Problema
O botão de ação principal ficava no rodapé — invisível sem scroll até o fim.
Ganho de UX
Botão primário sempre acessível. Contador de itens / volume total sempre visível.

06

Animação de atenção no estado vazio (hint pulse)

@keyframes hintPulse
Problema
Operadores novos não sabiam por onde começar. Tela em branco não comunicava o próximo passo.
Ganho de UX
Direcionamento visual imediato sem tooltip. A animação para no primeiro item adicionado — não irrita experientes.
Don Norman · signifiers Duolingo

07

Indicador de status na borda esquerda

border-left: 4px solid <cor-status>
Problema
Com múltiplos cards na tela, era difícil identificar quais itens ainda precisavam de atenção.
Ganho de UX
Scanning visual rápido — o operador varre a borda e identifica o status. Não depende de leitura de texto.

08

Filtros avançados colapsáveis

Progressive disclosure · toggle ▼/▲

Ganho de UX
Casual vê interface limpa. Avançado expande quando precisa. Complexidade disponível mas não imposta.

09

Campo de busca com autocomplete

debounce 250ms · max 6 resultados
Problema
Operadores digitavam códigos longos (SAP, CNPJ) de memória. Erros de digitação retornavam "sem resultados".
Ganho de UX
Recognition over recall (Nielsen #6). O sistema apresenta opções — o operador reconhece em vez de lembrar.
Nielsen Heuristic #6 Google Search
